להלן **הסבר ברור ומסודר על הספרייה `strings.h`**, כולל ההבדל בינה לבין `string.h`, ומה יש בה בפועל (כמקובל בלינוקס / Unix).

---

## מהי `strings.h`?

`strings.h` היא **ספריית עזר למחרוזות** שמקורה במערכות **Unix / BSD**, ונמצאת גם בלינוקס (glibc).

⚠️ חשוב:
**זו לא הספרייה הסטנדרטית של C** – התקן הרשמי הוא `string.h`.

---

## `string.h` לעומת `strings.h`

| `string.h`                | `strings.h`                      |
| ------------------------- | -------------------------------- |
| ספרייה סטנדרטית של C      | ספרייה לא־סטנדרטית (POSIX / BSD) |
| קיימת בכל קומפיילר C      | קיימת בעיקר בלינוקס/Unix         |
| פעולות בסיסיות על מחרוזות | פעולות משלימות / מיוחדות         |
| חובה למבחנים              | לפעמים מוזכרת כהרחבה             |

---

## מה יש ב־`strings.h`?

הספרייה מספקת פונקציות **שאינן חלק מתקן C**, בעיקר:

---

### `strcasecmp`

השוואת מחרוזות **בלי הבדל בין אותיות גדולות/קטנות**

```c
int strcasecmp(const char *s1, const char *s2);
```

דוגמה:

```c
strcasecmp("Hello", "hello") == 0   // true
```

---

### `strncasecmp`

כמו `strcasecmp`, אבל עד מספר תווים נתון

```c
int strncasecmp(const char *s1, const char *s2, size_t n);
```

---

### `bzero`

מאפס בלוק זיכרון (כיום נחשב מיושן)

```c
void bzero(void *s, size_t n);
```

שקול ל:

```c
memset(s, 0, n);
```

---

### `bcopy`

העתקת זיכרון (מיושן)

```c
void bcopy(const void *src, void *dst, size_t n);
```

שקול ל:

```c
memcpy(dst, src, n);
```

---

### `bcmp`

השוואת בלוקים בזיכרון

```c
int bcmp(const void *s1, const void *s2, size_t n);
```

שקול ל:

```c
memcmp(s1, s2, n);
```

---

## למה קיימת `strings.h` בכלל?

* תאימות לאחור (legacy Unix code)
* קוד ישן מאוד
* הרחבות BSD / POSIX

👉 בקוד חדש:

* משתמשים ב־`string.h`
* נמנעים מ־`bzero`, `bcopy`, `bcmp`
