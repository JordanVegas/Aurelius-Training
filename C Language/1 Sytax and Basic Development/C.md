ההבדל בין **`signed int`** ל־**`unsigned int`** נוגע ל־**האם ניתן לייצג מספרים שליליים** ומהו **טווח הערכים**.

---

## `signed int`

* יכול לייצג **מספרים חיוביים ושליליים**
* זהו טיפוס ברירת המחדל של `int`
* משתמש בביט אחד לסימן (±)

לדוגמה:

```c
signed int x = -10;
```

טווח אופייני (32-bit):

* ‎−2,147,483,648 עד ‎+2,147,483,647

---

## `unsigned int`

* יכול לייצג **מספרים לא־שליליים בלבד** (0 ומעלה)
* אין ביט סימן → כל הביטים לערך

לדוגמה:

```c
unsigned int y = 10;
```

טווח אופייני (32-bit):

* 0 עד ‎4,294,967,295

---

## הבדל עיקרי בקצרה

| signed int    | unsigned int    |
| ------------- | --------------- |
| כולל שליליים  | רק 0 ומעלה      |
| טווח קטן יותר | טווח גדול יותר  |
| ברירת מחדל    | דורש ציון מפורש |

---

## נקודה חשובה (מלכודת נפוצה)

השוואה בין signed ל־unsigned עלולה לגרום לבאגים:

```c
int a = -1;
unsigned int b = 1;

if (a < b) {
    // לא תמיד נכון כפי שמצפים
}
```

במקרה כזה `a` מומר ל־unsigned → נהיה מספר גדול מאוד.

---

> `signed int` מייצג מספרים שליליים וחיוביים, בעוד `unsigned int` מייצג רק מספרים לא־שליליים ולכן מאפשר טווח ערכים גדול יותר עבור אותו גודל זיכרון.
