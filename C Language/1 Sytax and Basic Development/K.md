**glibc** היא קיצור של **GNU C Library** – הספרייה הסטנדרטית של שפת C במערכות **Linux / Unix-like**.

---

## מה זה בעצם glibc?

**glibc היא ספריית מערכת**, שמספקת את המימוש בפועל של:

* פונקציות סטנדרטיות של C
* קריאות מערכת (system calls)
* שירותים בסיסיים של מערכת ההפעלה

היא מפותחת כחלק מפרויקט **GNU Project**.

---

## למה glibc חשובה?

כאשר אתה כותב קוד כזה:

```c
printf("Hello\n");
```

❗ הפונקציה `printf` **לא ממומשת בקומפיילר**
✔ היא ממומשת בתוך **glibc**

כלומר:

* הקומפיילר (GCC) → מקמפל את הקוד
* glibc → מספקת את המימוש בזמן הריצה

---

## מה glibc מספקת בפועל?

### פונקציות סטנדרטיות

* `printf`, `scanf`
* `malloc`, `free`
* `strlen`, `strcmp`
* `exit`, `atoi`

### גישה למערכת ההפעלה

* `open`, `read`, `write`
* `fork`, `exec`
* `time`, `sleep`

### ניהול תהליכים וזיכרון

* heap
* threads (pthread)
* signals

---

## מתי glibc נכנסת לפעולה?

בשלב **הריצה**, לא הקימפול.

רוב התוכניות בלינוקס הן:

* **dynamically linked**
* כלומר: נטענות יחד עם glibc בזמן הריצה

אפשר לראות זאת עם:

```bash
ldd program
```

---

## נקודה חשובה למבחן

* glibc ≠ GCC
* glibc ≠ kernel

| רכיב         | תפקיד       |
| ------------ | ----------- |
| GCC          | קומפיילר    |
| glibc        | ספריית C    |
| Linux Kernel | מערכת הפעלה |

---

## משפט קצר למבחן

> glibc היא ספריית ה־C הסטנדרטית של GNU, המספקת את המימוש של פונקציות C וקריאות מערכת, ומשמשת שכבת תיווך בין תוכנית המשתמש לקרנל.
להלן **סקירה יסודית וברורה על מערכים בשפת C**: איך מגדירים אותם, איך הם מיוצגים בזיכרון, ואיך ניגשים אליהם.
