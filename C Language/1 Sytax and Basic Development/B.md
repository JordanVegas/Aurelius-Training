שפת **C** היא שפת תכנות **פרוצדורלית**, וזה אומר משהו מאוד מסוים על **איך כותבים בה תוכניות ואיך חושבים בהן**.

---

## מה זה תכנות פרוצדורלי?

**תכנות פרוצדורלי** = תכנות שמבוסס על
➡️ **פירוק הבעיה לרצף של פעולות (procedures / functions)**
➡️ **הרצה סדרתית של פקודות**
➡️ **שימוש במשתנים שמשותפים לפונקציות**

כלומר:
התוכנית היא **רצף של צעדים**, לא אוסף של אובייקטים חכמים.

---

## איך זה נראה בפועל ב־C?

### 1️⃣ זרימה לינארית של הקוד

הקוד רץ **מלמעלה למטה**, שורה אחרי שורה:

```c
int x = 5;
x = x + 1;
printf("%d\n", x);
```

אין “התנהגות אוטונומית” של אובייקטים – הכול מתבצע לפי סדר הקריאה.

---

### 2️⃣ חלוקה לפונקציות (Procedures)

ב־C מפרקים את הבעיה לפונקציות, שכל אחת:

* מבצעת פעולה אחת
* מקבלת קלט
* מחזירה פלט (או לא)

```c
int sum(int a, int b) {
    return a + b;
}
```

ה־`main` הוא פשוט **פונקציה שמפעילה פונקציות אחרות**.

---

### 3️⃣ הנתונים מופרדים מהפעולות

בתכנות פרוצדורלי:

* הנתונים (variables, structs)
* והפעולות (functions)

הם **שני דברים נפרדים**

```c
int balance;

void deposit(int amount) {
    balance += amount;
}
```

אין “אובייקט חשבון” עם מתודות – רק נתונים ופונקציות שפועלות עליהם.

---

### 4️⃣ שליטה מלאה בזרימת התוכנית

השליטה נעשית בעזרת:

* `if / else`
* `for / while`
* `return`

```c
if (x > 0) {
    do_something();
} else {
    do_other();
}
```

אין מנגנונים כמו פולימורפיזם או dispatch דינמי.

---

### 5️⃣ אין מושגים של OOP

בשפת C **אין**:
❌ מחלקות
❌ אובייקטים
❌ ירושה
❌ encapsulation מובנה

אפשר לחקות חלקית עם `struct` ו־פונקציות, אבל זה לא מובנה בשפה.

---

## איך זה משפיע על אופן החשיבה?

כשמתכנתים ב־C שואלים:

* ❓ מה סדר הפעולות?
* ❓ איזו פונקציה עושה מה?
* ❓ איזה נתונים עוברים בין הפונקציות?

ולא:

* ❌ איזה אובייקט אחראי?
* ❌ מי יורש ממי?

---

## השוואה קצרה

| פרוצדורלי (C)             | מונחה עצמים (C++ / Java) |
| ------------------------- | ------------------------ |
| פונקציות                  | מחלקות                   |
| זרימה סדרתית              | אינטראקציה בין אובייקטים |
| נתונים גלובליים / מועברים | נתונים בתוך אובייקט      |
| שליטה ידנית               | הרבה אוטומציה            |

---

## משפט סיכום טוב

> שפת C היא שפת תכנות פרוצדורלית, כלומר התוכנית בנויה כרצף של פעולות ופונקציות הפועלות על נתונים, כאשר הדגש הוא על זרימת הביצוע והשליטה המפורשת בקוד, ולא על אובייקטים או יחסי ירושה.
